**마감**: 다음 주 화 17:00 (1분 영상 + 1page 기획서 + Notion URL + 트랙별 산출물)

## 1. 공통 컨텍스트 (복붙용)

두 모델에 붙여넣을 **공통 배경**을 먼저 넣으세요. 길어도 괜찮습니다.

```
[프로젝트 배경]
- 프로젝트명: AdStrategy AI (데이터 기반 디지털 광고 전략 설계)
- 데이터: Kaggle 원본 1,800건 → 4단계 파이프라인으로 10,030건×42피처 보강 (합성 데이터 ~46% 포함)
- 분석: ML 예측(ROAS/CPC/CPA), ANOVA·t-test·Chi²·SHAP, Ablation Study V1~V5
- 핵심 발견: Phase 3 데이터 생성 시 target leakage(bounce_rate, load_time 등) 발견 → leakage 제거 시 R² 0.79→0.40, "높은 R²의 함정"을 체계적 감사로 규명
- 제출 요구사항: 1분 브리핑 영상, 1page 기획서(PDF), 데이터분석 트랙 Notion 공개페이지(시각화·결론·핵심 이미지 포함)
- 평가: 상상력(10)·실행력(10)·영향력(10), "기능 나열이 아닌 문제 해결 과정과 결과" 강조
```

## 2. Gemini Pro에 넣을 요청 (2가지 시나리오)

### A. Gemini "심층 리서치" / 에이전트 모드

- **용도**: 웹/논문 기반 최신 사례 + 방법론 제안  
- **입력**: 아래 프롬프트를 그대로 복사해 붙여넣기. 필요하면 위 "[프로젝트 배경]" 블록을 그 위에 붙인 뒤 보내기.

```
[위 프로젝트 배경을 전제로]

다음 두 가지를 수행해줘.

1) **데이터 분석 트랙 경쟁력 강화 리서치**
- 디지털 광고·마케팅 분석, MMM(Media Mix Modeling), ROAS 예측 관련 논문·블로그·캐글 노트북 중에서,
  "평가자가 30초 만에 핵심을 파악하고 인사이트 가치를 높게 줄" 수 있는 **분석 기법·시각화·스토리텔링** 사례 3~5개를 찾아 요약해줘.
- 각 사례마다: 출처(URL 가능하면), 한 줄 요약, 우리 프로젝트에 적용 가능한 구체적 아이디어 1~2개.

2) **1주일 내 고도화 우선순위 제안**
- 현재 우리 프로젝트의 강점(ablation으로 leakage 규명, 합성 데이터 영향 검증)과 약점(합성 46%, honest R² 0.40~0.53)을 고려해서,
  "제출 마감 1주일 전에 할 수 있는" 개선 작업을 우선순위 순으로 5~7개 제안해줘.
  각 항목: 제목, 1~2문장 설명, 예상 효과(평가 기준: 상상력/실행력/영향력 중 어떤 것에 기여하는지), 난이도(상/중/하).

출력 형식: 마크다운으로 번호·소제목을 사용해 정리해줘. 나중에 ChatGPT 결과와 비교할 수 있게 구조화해줘.
```

- **팁**: Gemini에서 "Google 검색 기반 답변"이 있다면 켜두면 논문/블로그 출처가 풍부해짐.

데이터 분석 트랙에서 "영향력·인사이트" 점수를 올리기 위한 분석 고도화 아이디어 5개와, 1page 기획서에서 30초 만에 핵심이 전달되도록 하는 제목·구성 제안을 해줘.

## 3. ChatGPT Pro에 넣을 요청 (2가지 시나리오)

### A. GPT-4o / Pro "심층 분석" 요청

- **용도**: 구조화된 개선안 + 1page/영상 스크립트 초안  
- **입력**: 아래를 복사해 붙여넣기. 앞에 "[프로젝트 배경]" 붙이기.

```
[위 프로젝트 배경을 전제로]

당신은 데이터 분석 트랙 심사위원 겸 기획 전문가 역할이다. 다음을 수행해줘.

1) **제출물별 개선 포인트**
- 1 Page 기획서: "30초 만에 핵심 파악"을 위해 필요한 항목(주제, 가설, 데이터 출처, 분석 방법, 핵심 인사이트)별로, 우리 프로젝트에 맞는 **구체적 문장 예시** 1~2개씩 제안해줘.
- 1분 브리핑 영상: (0–10초) 배경, (10–25초) 분석 과정, (25–50초) 시각화, (50–60초) 인사이트 — 구간별 **나레이션 초안** (한국어, 실제 읽을 수 있는 길이).
- Notion 페이지: "기능 나열이 아닌 문제 해결 과정·결과"가 드러나도록 섹션 구성안과 각 섹션에 들어갈 핵심 문구 1줄씩.

2) **분석 고도화 제안**
- 기존 분석(ANOVA, t-test, SHAP, Ablation V1~V5)을 유지한 채, "실행력·영향력"을 높이기 위한 **추가 분석 또는 시각화** 3~5개를 제안해줘. 각각: 목적, 방법(예: Python/scipy·seaborn), 예상 결과, 제출물에서 어디에 쓸지(기획서/Notion/영상).

3) **심사 관점 반영**
- 평가 기준(상상력 10, 실행력 10, 영향력 10)에 맞춰, 우리 프로젝트의 **강점 2개, 보완할 점 2개**를 짧게 정리해줘.

출력은 마크다운으로, 번호와 소제목을 사용해 정리해줘.
```

## 6. 주의사항

- **프로젝트 배경**을 두 모델에 **동일하게** 주면, 제안을 나중에 비교·통합하기 쉽습니다.
- **"1~2문장 예시"**, **"나레이션 초안"**처럼 **바로 쓸 수 있는 형태**로 요청해 두면 기획서·스크립트 작성이 빨라집니다.
- Gemini·GPT 결과는 **참고용**으로만 쓰고, 최종 문장과 수치는 **반드시 실제 데이터·코드 결과와 맞춰서** Cursor에서 수정하세요.
- Notion URL은 **"웹에 게시"** 설정 후 제출 전 한 번 더 열어서 권한·이미지 로딩을 확인하세요.

이 가이드대로 요청한 뒤, 나온 결과를 `Gemini_리서치_결과.md`, `GPT_리서치_결과.md`로 저장해 두면, 다음 단계에서 Cursor로 "이 제안들 기준으로 1page 문단 초안/Notion 섹션 초안 만들어줘"처럼 이어서 요청할 수 있습니다.
